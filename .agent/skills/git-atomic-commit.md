# 🚀 작업 분류 및 커밋 지침

에이전트는 `git commit` 요청을 받으면 다음 절차를 엄격히 준수합니다.

## 1. 분석 단계
- `git status`와 `git diff`를 실행하여 모든 변경 사항을 확인합니다.
- 변경된 파일들을 다음 기준에 따라 **논리적 그룹**으로 분류합니다:
	- `feat`: 새로운 기능 추가 관련 파일들
	- `fix`: 버그 수정 관련 파일들
	- `refactor`: 코드 구조 개선 관련 파일들
	- `docs` 또는 `style`: 문서 및 포맷팅 변경

## 2. 실행 단계
- 분류된 그룹별로 `git add [파일명]`을 수행합니다.
- 각 그룹에 대해 독립적인 커밋 메시지를 작성하고 `git commit`을 실행합니다.
- **주의:** 절대 `git add .` 후 한 번에 커밋하지 마십시오.

## 3. 보고
- 수행 완료 후, 어떤 파일들이 어떤 그룹으로 분류되어 커밋되었는지 사용자에게 요약 보고합니다.
