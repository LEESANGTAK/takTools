
# 📜 Project Global Rules

## 💬 커뮤니케이션 규칙
- Implementation Plan 과 Task 등 artifacts 는 한국어를 기본 언어로 사용함.
-  사용자의 지시가 구체적이지 않거나 제안을 요청할 경우, 최대 세 가지의 안을 제시해 사용자가 바른 결정을 내도록 돕습니다.
- 수행하려는 작업의 이유와 방법을 먼저 설명하고, 사용자의 리뷰 및 승인을 거친 후 작업을 시작합니다.
- 이해를 돕기 위해 표, 이미지, 관련 링크를 적극 활용할 것.
- 답변 마지막에는 항상 핵심 요약을 포함할 것.

## 📏 코드 규모 관리 규정

에이전트는 파일을 생성하거나 기존 코드를 수정할 때 다음 규칙을 적용합니다.

### 1. 상시 체크 (1000줄 규칙)
- 수정 중인 파일의 전체 라인 수를 확인합니다.
- 만약 특정 파일이 **1000줄(1,000 lines)**에 도달하거나 초과할 경우, 작업을 일시 중단하고 사용자에게 알립니다.

### 2. 리팩토링 검토 및 제안
- 파일이 너무 커진 경우, 다음 방법 중 최적의 안을 구상하여 제안합니다:
	- **기능 분리:** 클래스나 모듈을 역할에 따라 2개 이상으로 분리.
	- **함수 추출:** 비대해진 로직을 별도의 헬퍼 클래스나 유틸리티로 이동.
	- **인터페이스 활용:** 추상화를 통해 의존성 분산.

### 3. 사용자 확인
- "파일이 1000줄을 초과했습니다. 리팩토링을 진행할까요?"라고 묻고, 사용자가 승인한 경우에만 분리 작업을 시작합니다.
